GitHub API 代理服务

由于浏览器安全限制，需要使用代理服务来调用 GitHub API

解决方案选项：

1. 使用 GitHub Actions Web 界面（推荐）

2. 部署 Vercel/Netlify Functions

3. 使用 GitHub CLI

方案 1: GitHub Actions Web 界面

1. 前往你的 GitHub 仓库

2. 点击 Actions 标签页

3. 选择 "Update V2Ray Config" workflow

4. 点击 "Run workflow" 按钮

5. 输入 Base64 编码的配置数据

方案 2: Vercel Function 示例

export default async function handler(req, res) { if (req.method !== 'POST') { return res.status(405).json({ error: 'Method not allowed' }); } const { repo, token, configData, commitMessage } = req.body; try { // 触发 GitHub Actions workflow const response = await fetch(`https://api.github.com/repos/${repo}/actions/workflows/update-v2.yml/dispatches`, { method: 'POST', headers: { 'Authorization': `token ${token}`, 'Accept': 'application/vnd.github.v3+json', 'Content-Type': 'application/json', }, body: JSON.stringify({ ref: 'main', inputs: { config_data: configData, commit_message: commitMessage || '🔄 Auto update v2 config' } }) }); if (response.ok) { res.status(200).json({ success: true, message: 'Workflow triggered successfully' }); } else { res.status(response.status).json({ error: 'Failed to trigger workflow' }); } } catch (error) { res.status(500).json({ error: error.message }); } }

方案 3: GitHub CLI 命令

# 安装 GitHub CLI # https://cli.github.com/ # 登录 gh auth login # 触发 workflow gh workflow run update-v2.yml \ --repo username/repository \ --field config_data="你的Base64编码配置" \ --field commit_message="更新配置"
返回转换器 查看 Actions
